About Dew-Drops

Find your inner dew

Dew-Drops is an airdrop platform for social media tasks. Campaigns create tasks (e.g. follow on Twitter, join Discord), build a merkle tree of eligible participants, and fund a pool. You prove eligibility with a merkle proof and claim your reward on-chain.

Task types

Tasks are tagged by kind: Twitter, Discord, Telegram, retweet, quote, like, comment, join, share, watch, follow, or custom. Each task has a reward amount and an end block.

Peak.tech & Pura

Peak.tech is the web interface for Dew-Drops. Use the Pura CLI app to build merkle trees, check eligibility, and submit claims from the command line.

FAQ

What is a merkle proof?

A short proof that your address (and proof nonce) is in the campaign's list without revealing the full list. The contract verifies it against the stored merkle root.

What is a proof nonce?

A unique value per participant per task so each address can only claim once. It's included in the leaf hash with your address and task ID.

Can I claim multiple tasks at once?

Yes. The DewDrops contract supports claimDropletBatch for multiple tasks in one transaction (up to the contract's MAX_CLAIM_BATCH).

What if a task has vesting?

You first call claimDropletVested to lock in your eligibility; rewards then vest over time. Use claimVested (or Pura claim-vested) to pull claimable amounts.

Dew-Drops uses a single contract (DewDrops.sol) for task creation, pool funding, and claim verification via merkle proofs. Vesting is optional per task.

  • · Leaf = keccak256(abi.encodePacked(participant, proofNonce, taskId, DOMAIN_SEED)).

Quick reference: claim flow

  1. Guardian creates task with merkle root and funds pool (topPool).
  2. Participants receive proof (address + proofNonce + merkleProof) from campaign.
  3. Participant calls claimDroplet(taskId, proofNonce, merkleProof) or uses Pura claim command.
  4. Contract verifies proof and sends reward in ETH.

Vesting flow

If a task has vesting enabled, use claimDropletVested instead of claimDroplet. Your reward is then locked and vests over blocks. Call claimVested(taskId) to withdraw claimable amounts as they unlock.
